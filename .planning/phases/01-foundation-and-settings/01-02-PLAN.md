**File 1: `admin/class-s3mo-settings-page.php`**

Class `S3MO_Settings_Page` with constructor accepting `?S3MO_Client $client` (nullable -- null when credentials missing).

Method `register_hooks()` -- registers four hooks:
- `add_action('admin_menu', [$this, 'add_menu'])`
- `add_action('admin_init', [$this, 'register_settings'])`
- `add_action('wp_ajax_s3mo_test_connection', [$this, 'ajax_test_connection'])`
- `add_action('admin_enqueue_scripts', [$this, 'enqueue_assets'])`

Method `add_menu()`:
- `add_media_page('CT S3 Offloader Settings', 'S3 Offloader', 'manage_options', 'ct-s3-offloader', [$this, 'render_page'])`
- Store the return value (hook suffix) in a property `$this->hook_suffix` for use in `enqueue_assets`

Method `register_settings()`:
- `register_setting('s3mo_settings', 's3mo_path_prefix', [...])` with sanitize_callback `[$this, 'sanitize_path_prefix']`, type 'string', default 'wp-content/uploads'
- `register_setting('s3mo_settings', 's3mo_delete_local', [...])` with sanitize_callback `'rest_sanitize_boolean'`, type 'boolean', default false

Method `sanitize_path_prefix(string $value): string`:
- Apply `sanitize_text_field()`
- `trim($value, '/')` to remove leading/trailing slashes
- If empty after trimming, call `add_settings_error('s3mo_path_prefix', 'empty', 'Path prefix cannot be empty.')` and return `get_option('s3mo_path_prefix', 'wp-content/uploads')`
- Return sanitized value

Method `ajax_test_connection()`:
- `check_ajax_referer('s3mo_test_nonce')` as FIRST line
- `if (!current_user_can('manage_options'))` -> `wp_send_json_error(['message' => 'Unauthorized'], 403)`
- `if (!$this->client)` -> `wp_send_json_error(['message' => 'AWS credentials not configured in wp-config.php'])`
- Get result from `$this->client->test_connection()`
- If `$result['success']` -> `wp_send_json_success($result)`
- Else -> `wp_send_json_error($result)`

Method `enqueue_assets(string $hook)`:
- Compare `$hook` against `$this->hook_suffix` (set in add_menu). If no match, return early.
- `wp_enqueue_script('s3mo-admin', S3MO_PLUGIN_URL . 'assets/js/admin.js', ['jquery'], S3MO_VERSION, true)`
- `wp_localize_script('s3mo-admin', 's3moAdmin', ['ajaxUrl' => admin_url('admin-ajax.php'), 'nonce' => wp_create_nonce('s3mo_test_nonce')])`

Method `render_page()`:
- Wrap in `<div class="wrap">`
- `<h1>CT S3 Offloader Settings</h1>`
- **Credentials section** (display-only, NOT editable):
  - `<h2>AWS Credentials</h2>`
  - `<p class="description">Credentials are read from wp-config.php constants. <a href="https://developer.wordpress.org/advanced-administration/wordpress/wp-config/" target="_blank">Learn more</a></p>`
  - HTML table with 4 rows showing status of each constant:
    - S3MO_BUCKET: If `defined('S3MO_BUCKET')`, show `S3MO_BUCKET` value. Else show `<span style="color:red">Not defined</span>` (use a CSS class, NOT inline style -- add a `.s3mo-not-defined` class)
    - S3MO_REGION: Same pattern, show value or "Not defined"
    - S3MO_KEY: If defined, show `'••••' . substr(S3MO_KEY, -4)` (masked with last 4 chars). Else "Not defined"
    - S3MO_SECRET: If defined, show `'••••••••'` (fully masked). Else "Not defined"
    - S3MO_CDN_URL: If defined and non-empty, show value. Else show "Not defined (optional — will use direct S3 URL)"
  - **IMPORTANT**: Use a CSS class for the "not defined" styling, not inline styles. Add minimal styles via `wp_add_inline_style` or an `admin.css` file.

- **Connection test section**:
  - `<h2>Connection Test</h2>`
  - `<button type="button" class="button button-secondary" id="s3mo-test-connection">Test Connection</button>`

  - `<form method="post" action="options.php">`
  - `settings_fields('s3mo_settings')`
  - `settings_errors()` to display validation messages
  - Table with:
    - **S3 Path Prefix**: text input, name `s3mo_path_prefix`, value from `get_option('s3mo_path_prefix', 'wp-content/uploads')`, description "Prefix added to S3 object keys (e.g., wp-content/uploads)"
    - **Delete Local Files**: checkbox, name `s3mo_delete_local`, checked if `get_option('s3mo_delete_local', false)`, description "Remove local file after successful S3 upload (saves disk space)"
  - Submit button via `submit_button()`

**File 2: `assets/js/admin.js`**

IIFE wrapping jQuery:
```javascript
(function($) {
    'use strict';

    $(document).on('click', '#s3mo-test-connection', function(e) {
        e.preventDefault();
        var $btn = $(this);
        var $result = $('#s3mo-test-result');

        $btn.prop('disabled', true).text('Testing...');
        $result.html('').removeClass('notice notice-success notice-error');

        $.ajax({
            url: s3moAdmin.ajaxUrl,
            type: 'POST',
            data: {
                action: 's3mo_test_connection',
                _ajax_nonce: s3moAdmin.nonce
            },
            success: function(response) {
                var isSuccess = response.success;
                var message = isSuccess ? response.data.message : (response.data && response.data.message ? response.data.message : 'Unknown error');
                $result
                    .addClass('notice ' + (isSuccess ? 'notice-success' : 'notice-error'))
                    .html('<p>' + message + '</p>');
            },
            error: function() {
                $result
                    .addClass('notice notice-error')
                    .html('<p>Request failed. Check your network connection.</p>');
            },
            complete: function() {
                $btn.prop('disabled', false).text('Test Connection');
            }
        });
    });
})(jQuery);
```

Key implementation notes:
- The AJAX handler uses `wp_send_json_success()` and `wp_send_json_error()` which wrap responses in `{success: true/false, data: {...}}` format. The JS must access `response.data.message` not `response.message`.
- The `enqueue_assets` hook check MUST use the stored `$this->hook_suffix` from `add_media_page()` return value, NOT a hardcoded string. This prevents the pitfall documented in research.
- Credentials section is entirely display-only. Only `s3mo_path_prefix` and `s3mo_delete_local` go through the Settings API form.
- `check_ajax_referer` as first line of AJAX handler satisfies SEC-03.
- `manage_options` capability check satisfies SEC-02.
