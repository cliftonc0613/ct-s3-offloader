<task type="auto">
  <name>Task 2: Add SDK version pinning to bootstrap</name>
  <files>ct-s3-offloader.php</files>
  <action>
    Add a runtime SDK version check to ct-s3-offloader.php. Specifically:

    1. After the constants section (after line 20), add:
       `define('S3MO_EXPECTED_AWS_SDK_VERSION', '3.337.3');`

    2. After the SDK is loaded (after `require_once $sdk_autoloader;` on line 58), add a version check block:

       ```php
       // Verify SDK version matches expected
       if (defined('Aws\Sdk::VERSION') && version_compare(\Aws\Sdk::VERSION, S3MO_EXPECTED_AWS_SDK_VERSION, '!=')) {
           add_action('admin_notices', function (): void {
               $expected = S3MO_EXPECTED_AWS_SDK_VERSION;
               $actual = \Aws\Sdk::VERSION;
               echo '<div class="notice notice-warning"><p><strong>CT S3 Offloader:</strong> '
                  . "Expected AWS SDK v{$expected} but found v{$actual}. "
                  . 'The plugin may not work correctly with this SDK version.</p></div>';
           });
       }
       ```

    3. Also update S3MO_VERSION from '1.0.0' to '1.1.0' (line 17) and the plugin header Version from 1.0.0 to 1.1.0 (line 6) to reflect the new release.

    Keep the existing SDK-not-found notice as-is. The version check is an additional safeguard, not a replacement. The plugin should continue to load even if versions mismatch — the notice is a warning, not a blocker.
  </action>
  <verify>
    - `grep 'S3MO_EXPECTED_AWS_SDK_VERSION' ct-s3-offloader.php` returns the constant definition
    - `grep 'Aws\\Sdk::VERSION' ct-s3-offloader.php` returns the version comparison
    - `grep "Version: 1.1.0" ct-s3-offloader.php` confirms version bump
    - `php -l ct-s3-offloader.php` passes syntax check (use system PHP)
  </verify>
  <done>Bootstrap file has SDK version pinning constant, runtime version mismatch warning, and version bumped to 1.1.0</done>
</task>
