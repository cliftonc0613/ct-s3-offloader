**Create `includes/class-s3mo-stats.php`:**

Class `S3MO_Stats` with static methods (same pattern as S3MO_Tracker):

1. `public static function calculate(): array` — Queries the database to build stats. Returns:
   ```php
   [
       'total_files'    => int,   // count of attachments with _s3mo_offloaded = '1'
       'total_size'     => int,   // sum of filesize from wp_get_attachment_metadata for offloaded attachments (bytes)
       'pending'        => int,   // total attachments minus offloaded
       'last_offloaded' => string // most recent _s3mo_offloaded_at value, or empty string
   ]
   ```

   For `total_files` and `pending`: Use a WP_Query with `meta_query` for `_s3mo_offloaded = '1'` to get offloaded count. Use a second WP_Query without meta filter to get total attachment count. `pending = total - offloaded`.

   For `total_size`: Loop through offloaded attachment IDs (process in batches of 100 for memory). For each, get `wp_get_attachment_metadata()` and sum `$metadata['filesize']` if it exists. If `filesize` key is missing (older WP), fall back to the file's `sizes` array and sum widths (skip — just use 0 for that file). Do NOT call `filesize()` on local paths or S3 API. This is a best-effort estimate from metadata.

   For `last_offloaded`: Query `$wpdb->postmeta` directly: `SELECT meta_value FROM {$wpdb->postmeta} WHERE meta_key = '_s3mo_offloaded_at' ORDER BY meta_value DESC LIMIT 1`.

2. `public static function get_cached(): array` — Returns cached stats from transient `s3mo_stats_cache` (1 hour expiry). If transient is empty, calls `calculate()` and sets transient.

3. `public static function refresh(): array` — Deletes transient `s3mo_stats_cache`, calls `calculate()`, sets new transient, and returns the result.

```javascript
$(document).on('click', '#s3mo-refresh-stats', function(e) {
    e.preventDefault();
    var $btn = $(this);
    var $status = $('#s3mo-stats-status');

    $btn.prop('disabled', true).text('Refreshing...');
    $status.text('');

    $.ajax({
        url: s3moAdmin.ajaxUrl,
        type: 'POST',
        data: {
            action: 's3mo_refresh_stats',
            _ajax_nonce: s3moAdmin.nonce
        },
        success: function(response) {
            if (response.success) {
                $('#s3mo-stat-files').text(response.data.total_files);
                $('#s3mo-stat-size').text(response.data.total_size);
                $('#s3mo-stat-pending').text(response.data.pending);
                $('#s3mo-stat-last').text(response.data.last_offloaded);
                $status.text('Updated!');
            } else {
                $status.text('Refresh failed.');
            }
        },
        error: function() {
            $status.text('Request failed.');
        },
        complete: function() {
            $btn.prop('disabled', false).text('Refresh Stats');
        }
    });
});
```

**Rewrite `uninstall.php`:**

The current file only deletes 2 options and 1 meta key. Expand it to remove ALL plugin data.

```php
<?php
/**
 * CT S3 Offloader — Uninstall
 *
 * Removes all plugin data when deleted through the WordPress admin.
 * Optionally deletes S3 objects if the setting is enabled.
 */

defined('WP_UNINSTALL_PLUGIN') || exit;

/* ─── Optional S3 object deletion ─────────────────────────────────── */

$delete_s3 = get_option('s3mo_delete_s3_on_uninstall', false);

if ($delete_s3) {
    // Load AWS SDK and plugin classes for S3 deletion.
    $plugin_dir = plugin_dir_path(__FILE__);
    $sdk_autoloader = $plugin_dir . 'aws-sdk/aws-autoloader.php';

    if (
        file_exists($sdk_autoloader)
        && defined('S3MO_BUCKET')
        && defined('S3MO_REGION')
        && defined('S3MO_KEY')
        && defined('S3MO_SECRET')
    ) {
        if (! class_exists('Aws\Sdk')) {
            require_once $sdk_autoloader;
        }

        require_once $plugin_dir . 'includes/class-s3mo-client.php';
        require_once $plugin_dir . 'includes/class-s3mo-tracker.php';

        try {
            $client = new S3MO_Client();
            $prefix = get_option('s3mo_path_prefix', 'wp-content/uploads');

            // Query all offloaded attachments.
            $args = [
                'post_type'      => 'attachment',
                'post_status'    => 'inherit',
                'fields'         => 'ids',
                'no_found_rows'  => true,
                'posts_per_page' => 100,
                'orderby'        => 'ID',
                'order'          => 'ASC',
                'meta_query'     => [
                    [
                        'key'     => '_s3mo_offloaded',
                        'value'   => '1',
                        'compare' => '=',
                    ],
                ],
            ];

            $page = 1;
            do {
                $args['paged'] = $page;
                $query = new WP_Query($args);
                $ids = array_map('intval', $query->posts);

                foreach ($ids as $attachment_id) {
                    $s3_key = get_post_meta($attachment_id, '_s3mo_key', true);
                    if (! empty($s3_key)) {
                        $client->delete_object($s3_key);
                    }

                    // Also delete thumbnail keys.
                    $metadata = wp_get_attachment_metadata($attachment_id);
                    if (! empty($metadata['sizes']) && is_array($metadata['sizes'])) {
                        $subdir = dirname($metadata['file']);
                        foreach ($metadata['sizes'] as $size_data) {
                            $thumb_key = $prefix . '/' . $subdir . '/' . $size_data['file'];
                            $client->delete_object($thumb_key);
                        }
                    }
                }

                $page++;
                wp_cache_flush();
            } while (count($ids) === 100);

        } catch (Exception $e) {
            // Log but don't halt uninstall.
            error_log('CT S3 Offloader uninstall: S3 deletion error — ' . $e->getMessage());
        }
    }
}

/* ─── Remove all postmeta ─────────────────────────────────────────── */

// Use delete_post_meta_by_key() for proper cache invalidation.
delete_post_meta_by_key('_s3mo_offloaded');
delete_post_meta_by_key('_s3mo_key');
delete_post_meta_by_key('_s3mo_bucket');
delete_post_meta_by_key('_s3mo_offloaded_at');

/* ─── Remove all options ──────────────────────────────────────────── */

delete_option('s3mo_delete_local');
delete_option('s3mo_path_prefix');
delete_option('s3mo_delete_s3_on_uninstall');

/* ─── Remove all transients ───────────────────────────────────────── */

delete_transient('s3mo_connection_status');
delete_transient('s3mo_stats_cache');

/* ─── Remove log file ─────────────────────────────────────────────── */

$log_file = plugin_dir_path(__FILE__) . 'ct-s3-migration.log';
if (file_exists($log_file)) {
    @unlink($log_file);
}
```

- Deletes ALL 4 meta keys (was only deleting `_s3mo_offloaded`)
- Deletes ALL 3 options (was only deleting 2, missing `s3mo_delete_s3_on_uninstall`)
- Deletes both transients
- Removes log file
- Optionally deletes S3 objects in batches if setting is enabled
