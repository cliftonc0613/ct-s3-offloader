**Create `admin/class-s3mo-media-column.php`:**

Class `S3MO_Media_Column` with a `register_hooks()` method that adds:

1. `manage_media_columns` filter — adds an "Offload" column header. Column key: `s3mo_status`.
2. `manage_media_custom_column` action — renders the column content for each attachment using `S3MO_Tracker::get_offload_info($post_id)`.
3. `admin_enqueue_scripts` action — enqueues `assets/css/admin.css` on all admin pages (the column appears on `upload.php`). Scope to admin pages that need it: check `$hook` is `upload.php` OR the settings page hook suffix (accept it as constructor param or check broadly for upload.php).

Column rendering logic:
- Call `S3MO_Tracker::get_offload_info($post_id)` to get offload data.
- If `offloaded === '1'`: render green dot + "S3" label. Wrap in a `<button>` with class `s3mo-status-toggle` and `data-id="{$post_id}"`. Below it, render a hidden `<div class="s3mo-details" id="s3mo-details-{$post_id}">` containing: S3 Key (the `key` value), Bucket (`bucket` value), and Uploaded (`offloaded_at` formatted with `human_time_diff(strtotime($offloaded_at), current_time('timestamp')) . ' ago'`).
- If `offloaded` is empty/falsy: render yellow dot + "Local" label. No popup needed.
- Error state: For now, there is no `_s3mo_error` meta. Show "Local" for non-offloaded items. (Error tracking can be added later if needed.)

Dot rendering: Use a `<span class="s3mo-status-dot s3mo-status-{state}"></span>` where state is `s3` or `local`.

**Create `admin/class-s3mo-admin-notices.php`:**

Class `S3MO_Admin_Notices` with a `register_hooks()` method that hooks `admin_notices` at priority 10.

The render method checks these conditions and outputs notices:
1. **Missing credentials** — check if any of `S3MO_BUCKET`, `S3MO_REGION`, `S3MO_KEY`, `S3MO_SECRET` are undefined. Output `notice-warning` with list of missing constants. This replaces the inline anonymous function currently in `ct-s3-offloader.php`.
2. **Failed connection test** — check transient `s3mo_connection_status`. If it exists and its value is falsy/error, output `notice-error` saying "S3 connection test failed. Check your credentials and bucket configuration."
3. **Missing CDN URL** — if `S3MO_CDN_URL` is not defined or empty, output `notice-info` (not error) saying CloudFront URL is not configured and files will use direct S3 URLs.

All notices are dismissible: add `is-dismissible` class. They reappear on next page load because we do NOT persist dismissal state.

Only show notices to users with `manage_options` capability.
