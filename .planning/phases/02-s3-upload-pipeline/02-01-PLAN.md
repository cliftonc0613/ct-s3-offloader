<task type="auto">
  <name>Task 1: Add upload_object, delete_object, and get_object_url to S3MO_Client</name>
  <files>includes/class-s3mo-client.php</files>
  <action>
Add three new methods to the existing S3MO_Client class. Add `use Aws\S3\ObjectUploader;` at the top alongside the existing use statements.

**Method 1: upload_object(string $key, string $file_path, string $content_type): array**

- Open file with `$body = fopen($file_path, 'rb')` — throw RuntimeException if fopen returns false
- Create ObjectUploader: `new ObjectUploader($this->s3, $this->bucket, $key, $body, 'private', ['params' => ['ContentType' => $content_type, 'CacheControl' => 'public, max-age=31536000, immutable']])`
- Call `$result = $uploader->upload()`
- Always fclose($body) in a finally block
- Return `['success' => true, 'key' => $key, 'url' => $this->get_object_url($key)]`
- Catch AwsException: return `['success' => false, 'key' => $key, 'error' => $e->getAwsErrorMessage() ?? $e->getMessage()]`

IMPORTANT: ACL MUST be 'private' — CloudFront OAC handles access, never use 'public-read'. This satisfies CDN-04.
IMPORTANT: CacheControl 'public, max-age=31536000, immutable' satisfies CDN-02.

**Method 2: delete_object(string $key): array**

- Call `$this->s3->deleteObject(['Bucket' => $this->bucket, 'Key' => $key])`
- Return `['success' => true, 'key' => $key]`
- Catch AwsException: return `['success' => false, 'key' => $key, 'error' => $e->getAwsErrorMessage() ?? $e->getMessage()]`

**Method 3: get_object_url(string $key): string**

- Return `$this->get_url_base() . '/' . ltrim($key, '/')`
- This leverages the existing get_url_base() which already handles CloudFront vs direct S3 URL selection.

<task type="auto">

  <files>includes/class-s3mo-tracker.php</files>
  <action>
Create a new file `includes/class-s3mo-tracker.php` with class S3MO_Tracker. This class manages postmeta for tracking which attachments have been offloaded to S3.

Meta keys (consistent with uninstall.php which already deletes `_s3mo_offloaded`):
- `_s3mo_offloaded` — boolean flag (1 or 0)
- `_s3mo_key` — the S3 object key for the original file
- `_s3mo_bucket` — bucket name at time of upload
- `_s3mo_offloaded_at` — ISO 8601 timestamp

**Static methods (all static, no instance state needed):**

**mark_as_offloaded(int $attachment_id, string $s3_key, string $bucket): void**
- update_post_meta($attachment_id, '_s3mo_offloaded', '1')
- update_post_meta($attachment_id, '_s3mo_key', $s3_key)
- update_post_meta($attachment_id, '_s3mo_bucket', $bucket)
- update_post_meta($attachment_id, '_s3mo_offloaded_at', gmdate('c'))

**is_offloaded(int $attachment_id): bool**
- return (bool) get_post_meta($attachment_id, '_s3mo_offloaded', true)

**get_s3_key(int $attachment_id): string**
- return (string) get_post_meta($attachment_id, '_s3mo_key', true)

**get_offload_info(int $attachment_id): array**
- Return associative array with all four meta values (for admin display in Phase 6)

**clear_offload_status(int $attachment_id): void**
- delete_post_meta for all four keys
- (Used by Phase 4 deletion sync and Phase 5 re-offload scenarios)

Add proper PHPDoc, `defined('ABSPATH') || exit;` guard at top. No constructor needed — all methods are static.
