<objective>
Create the S3MO_URL_Rewriter class with the two foundational URL rewriting hooks: wp_get_attachment_url (canonical attachment URL) and the_content (post/page content replacement). Wire it into the plugin bootstrap.

Purpose: These two hooks cover the majority of URL rewriting — attachment URLs power most WordPress media functions, and the_content filter handles inline media in posts/pages (including Gutenberg block output). The REST API's content.rendered field also applies the_content filters, so this plan provides partial headless frontend support automatically.

Output: S3MO_URL_Rewriter class file and updated bootstrap in ct-s3-offloader.php
</objective>

<task type="auto">
  <name>Task 1: Create S3MO_URL_Rewriter class with attachment URL and content filters</name>
  <files>includes/class-s3mo-url-rewriter.php</files>
  <action>
Create `includes/class-s3mo-url-rewriter.php` with the S3MO_URL_Rewriter class.

Constructor:
- Accepts S3MO_Client instance via dependency injection and stores it as private property.

register_hooks() method:
- Hooks `wp_get_attachment_url` filter at priority 10 with 2 args (url, attachment_id).
- Hooks `the_content` filter at priority 10 with 1 arg.

filter_attachment_url($url, $attachment_id) method:
- Guard: if !S3MO_Tracker::is_offloaded($attachment_id), return $url unchanged.
- Get S3 key via S3MO_Tracker::get_s3_key($attachment_id).
- Guard: if empty key, return $url unchanged.
- Return $this->client->get_object_url($key).
- This is safe against double-rewriting because get_object_url always produces the same URL.

filter_content($content) method:
- Guard: if empty($content), return $content.
- Get the local upload base URL via wp_get_upload_dir()['baseurl']. Never hardcode paths.
- Get the CDN base URL via $this->client->get_url_base().
- Guard: if local base equals CDN base (unlikely but safe), return $content.
- Build replacements array with both http:// and https:// variants of the local base URL. Do this by checking if local base starts with https, then also add http variant, and vice versa. This handles mixed-protocol content stored in the database.
- Use str_replace (NOT regex) to replace all local base URL variants with CDN base URL + '/wp-content/uploads' (or the configured path prefix from get_option('s3mo_path_prefix', 'wp-content/uploads')).
- Return the modified content.

Private helper get_cdn_uploads_base() method:
- Returns $this->client->get_url_base() . '/' . ltrim(get_option('s3mo_path_prefix', 'wp-content/uploads'), '/').
- Cache in a private property to avoid repeated option lookups.

Important patterns:
- Use str_replace, NOT preg_replace — simpler, faster, and sufficient for URL replacement.
- NEVER modify database values — this is runtime-only filtering.
- wp_get_upload_dir()['baseurl'] gives the correct local URL base — never hardcode /wp-content/uploads.
- Always check is_offloaded() before rewriting individual attachment URLs.
- Content filter does bulk str_replace (no per-attachment check needed — replaces upload base path).
