  <name>Task 1: Add status/reset query helpers to S3MO_Bulk_Migrator</name>
  <files>includes/class-s3mo-bulk-migrator.php</files>
  <action>
Add three new methods to the existing `S3MO_Bulk_Migrator` class:

1. `public function get_status_counts(?string $mime_type = null): array`
   - Query total attachments: `get_posts()` with `post_type => 'attachment'`, `post_status => 'inherit'`, `fields => 'ids'`, `posts_per_page => -1`, `no_found_rows => true`. If `$mime_type` set, add `post_mime_type`.
   - Query offloaded attachments: same query plus `meta_query` where `_s3mo_offloaded` = '1'.
   - Calculate pending = total - offloaded.
   - Return `['total' => $total, 'offloaded' => $offloaded, 'pending' => $pending]`

2. `public function get_all_attachment_statuses(?string $mime_type = null): array`
   - Query ALL attachments (same base query as get_status_counts, `posts_per_page => -1`).
   - For each attachment ID, build an array item:
     ```php
     [
         'ID'       => $id,
         'Filename' => basename(get_post_meta($id, '_wp_attached_file', true) ?: '(unknown)'),
         'MIME'     => get_post_mime_type($id),
         'Status'   => S3MO_Tracker::is_offloaded($id) ? 'offloaded' : 'pending',
         'S3 Key'   => S3MO_Tracker::get_s3_key($id) ?: '-',
     ]
     ```
   - Process in batches of 100 with `cleanup_memory()` between batches to handle large libraries.
   - Return the full array of items.

3. `public function reset_tracking(?string $mime_type = null, bool $delete_remote = false): array`
   - Query offloaded attachments: `get_posts()` with `meta_query` where `_s3mo_offloaded = '1'`, `posts_per_page => -1`, `fields => 'ids'`. If `$mime_type` set, add `post_mime_type`.
   - Initialize counters: `$cleared = 0`, `$deleted = 0`, `$delete_errors = 0`.
   - For each attachment ID:
     - If `$delete_remote`: get S3 key via `S3MO_Tracker::get_s3_key($id)`, call `$this->client->delete_object($key)`. If success, increment `$deleted`. If fail, increment `$delete_errors` and log with `error_log()`. Also delete all thumbnail keys (get metadata, build key list same as upload logic, delete each).
     - Call `S3MO_Tracker::clear_offload_status($id)`, increment `$cleared`.
   - Call `cleanup_memory()` every 100 items.
   - Return `['cleared' => $cleared, 'deleted' => $deleted, 'delete_errors' => $delete_errors]`

Follow existing code style in the file. PHPDoc on every method.

<task type="auto">
  <name>Task 2: Implement status and reset subcommands in S3MO_CLI_Command</name>
  <files>includes/class-s3mo-cli-command.php</files>
  <action>
Replace the stub `status()` and `reset()` methods in `S3MO_CLI_Command` with full implementations.

**status() method:**

WP-CLI PHPDoc header with options:
- `[--verbose]` : Show per-file table instead of summary
- `[--mime-type=<type>]` : Filter by MIME type
- `[--format=<format>]` : Output format (default: table, options: table, csv)
- `@when after_wp_load`

Implementation:
1. Parse args:
   ```php
   $verbose   = \WP_CLI\Utils\get_flag_value($assoc_args, 'verbose', false);
   $mime_type = $assoc_args['mime-type'] ?? null;
   $format    = $assoc_args['format'] ?? 'table';
   ```

2. **Summary mode (default):**
   - Call `$this->migrator->get_status_counts($mime_type)`
   - Display as formatted table using `\WP_CLI\Utils\format_items()`:
     ```php
     $items = [
         ['Metric' => 'Total attachments', 'Count' => $counts['total']],
         ['Metric' => 'Offloaded to S3',   'Count' => $counts['offloaded']],
         ['Metric' => 'Pending',           'Count' => $counts['pending']],
     ];
     \WP_CLI\Utils\format_items($format, $items, ['Metric', 'Count']);
     ```
   - If all offloaded: `WP_CLI::success('All files are offloaded to S3.')`
   - If pending > 0: `WP_CLI::log(sprintf('Run "wp ct-s3 offload" to upload %d pending files.', $counts['pending']))`

3. **Verbose mode (`--verbose`):**
   - Call `$this->migrator->get_all_attachment_statuses($mime_type)`
   - Display with `\WP_CLI\Utils\format_items($format, $items, ['ID', 'Filename', 'MIME', 'Status', 'S3 Key'])`
   - Show summary line after table: total, offloaded, pending counts

**reset() method:**

WP-CLI PHPDoc header with options:
- `[--delete-remote]` : Also delete S3 objects
- `[--mime-type=<type>]` : Filter by MIME type
- `[--yes]` : Skip confirmation prompt
- `@when after_wp_load`

Implementation:
1. Parse args:
   ```php
   $delete_remote = \WP_CLI\Utils\get_flag_value($assoc_args, 'delete-remote', false);
   $mime_type     = $assoc_args['mime-type'] ?? null;
   ```

2. Build confirmation message:
   - If `$delete_remote`: `'This will clear ALL offload tracking AND delete ALL S3 objects. Continue?'`
   - Else: `'This will clear ALL offload tracking metadata. Continue?'`
   - If `$mime_type`: append ` (filtered to {$mime_type})` to message

3. Call `WP_CLI::confirm($message, $assoc_args)` -- this respects `--yes` flag automatically.

4. Call `$this->migrator->reset_tracking($mime_type, $delete_remote)`

5. Display results:
   - `WP_CLI::success(sprintf('Cleared tracking for %d attachments.', $result['cleared']))`
   - If `$delete_remote && $result['deleted'] > 0`: `WP_CLI::log(sprintf('%d S3 objects deleted.', $result['deleted']))`
   - If `$result['delete_errors'] > 0`: `WP_CLI::warning(sprintf('%d S3 deletions failed (see error log).', $result['delete_errors']))`

Also update the class-level PHPDoc to include all three subcommands in the EXAMPLES section.

Do NOT use `WP_CLI::error()` for recoverable issues. Only use it if credentials are missing (but that's already handled by bootstrap).
  </action>
