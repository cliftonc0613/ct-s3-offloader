Create the bulk migration engine and WP-CLI `offload` command that uploads an existing media library to S3 in configurable batches with retry, progress output, dry-run mode, and fault tolerance.

Purpose: Site owners with existing media libraries need a CLI tool to migrate thousands of files to S3 without manual intervention, with the ability to resume after failures.
Output: Two new classes (`S3MO_Bulk_Migrator` and `S3MO_CLI_Command`) plus bootstrap wiring, delivering a working `wp ct-s3 offload` command.

  <name>Task 1: Create S3MO_Bulk_Migrator batch processing engine</name>
  <files>includes/class-s3mo-bulk-migrator.php</files>
  <action>
Create `includes/class-s3mo-bulk-migrator.php` with class `S3MO_Bulk_Migrator`.

Constructor takes `S3MO_Client $client`. Store as private property.

**Methods to implement:**

1. `public function count_attachments(?string $mime_type, bool $force): int`
   - Run a `get_posts()` query with `post_type => 'attachment'`, `post_status => 'inherit'`, `fields => 'ids'`, `no_found_rows => true`, `posts_per_page => -1`
   - When `$force` is false, add `meta_query` to exclude `_s3mo_offloaded = '1'` (use OR: NOT EXISTS or != '1')
   - When `$mime_type` is non-null, set `post_mime_type`
   - Return `count()` of the result array

2. `public function get_next_batch(int $batch_size, ?string $mime_type, bool $force): array`
   - Same query as count but with `posts_per_page => $batch_size` (NOT -1)
   - `orderby => 'ID'`, `order => 'ASC'`
   - Returns `int[]` array of attachment IDs

3. `public function build_file_key_list(int $attachment_id): array`
   - **This is a shared helper used by both `upload_attachment` and Plan 02's `reset_tracking`.**
   - Get metadata with `wp_get_attachment_metadata($attachment_id)`
   - If empty `$metadata['file']`, return empty array
   - Build file list (original + thumbnails) using exact same pattern from `S3MO_Upload_Handler::handle_upload()`:
     - `$upload_dir = wp_get_upload_dir()`, `$prefix = get_option('s3mo_path_prefix', 'wp-content/uploads')`
     - Original: `['local' => $upload_dir['basedir'] . '/' . $metadata['file'], 'key' => $prefix . '/' . $metadata['file']]`
     - Thumbnails: if `$metadata['sizes']` is non-empty array, derive `$subdir = dirname($metadata['file'])`, then for each `$size_data` in `$metadata['sizes']`: `['local' => $upload_dir['basedir'] . '/' . $subdir . '/' . $size_data['file'], 'key' => $prefix . '/' . $subdir . '/' . $size_data['file']]`
   - Return the array of `['local' => ..., 'key' => ...]` items

4. `public function upload_attachment(int $attachment_id, int $max_retries = 2): array`
   - If already offloaded and NOT force mode, return `['status' => 'skip', 'error' => 'Already offloaded']`
   - Call `$files = $this->build_file_key_list($attachment_id)`
   - If empty `$files`, return `['status' => 'skip', 'error' => 'No file in metadata']`
   - Upload with retry loop: up to `$max_retries` retries (3 total attempts)
   - For each attempt, iterate all `$files` and call `$this->client->upload_object($file['key'], $file['local'], $content_type)` for each
   - Exponential backoff between retries: `sleep(pow(2, $attempt - 1))` giving 1s, 2s
   - **On full success (all files uploaded):** Extract the S3 key from the first file entry and call:
     ```php
     S3MO_Tracker::mark_as_offloaded(
         $attachment_id,
         $files[0]['key'],
         $this->client->get_bucket()
     );
     ```
     `mark_as_offloaded` requires exactly 3 arguments: `(int $attachment_id, string $s3_key, string $bucket)`. The S3 key is `$files[0]['key']` (the original file's key), and the bucket comes from `$this->client->get_bucket()`.
   - Return `['status' => 'success', 'files' => count($files)]`
   - On failure after all retries: return `['status' => 'fail', 'error' => $last_error]`

5. `public function cleanup_memory(): void`
   - Three-step cleanup:
     1. `wp_cache_flush()`
     2. `global $wpdb; $wpdb->queries = [];`
     3. `if (function_exists('gc_collect_cycles')) { gc_collect_cycles(); }`

6. `public function get_attachment_info(int $attachment_id): array`
   - Returns `['id' => $id, 'filename' => basename($metadata['file']), 'size' => filesize(...), 'mime' => get_post_mime_type($id)]`
   - Used by dry-run and progress output
   - Handle missing file gracefully (size = 0 if file not found locally)

Follow existing code style: PHPDoc on every method, `defined('ABSPATH') || exit;` at top, type hints on all parameters and return types.

  <action>
**Part A: Create `includes/class-s3mo-cli-command.php`**

Create class `S3MO_CLI_Command extends WP_CLI_Command`.

Private properties: `S3MO_Client $client`, `S3MO_Bulk_Migrator $migrator`.

Constructor: takes `S3MO_Client $client`, creates `$this->migrator = new S3MO_Bulk_Migrator($client)`.

**Implement `public function offload($args, $assoc_args)` method:**

Add the WP-CLI PHPDoc header with all options documented (copy exact format from 05-RESEARCH.md Pattern 2):
- `--dry-run`, `--force`, `--batch-size=<number>` (default 100), `--sleep=<seconds>` (default 0), `--mime-type=<type>`, `--limit=<number>`
- Include `@when after_wp_load`

Method body:

1. Parse arguments using `\WP_CLI\Utils\get_flag_value()` for boolean flags and direct array access with defaults for valued args:
   ```php
   $dry_run    = \WP_CLI\Utils\get_flag_value($assoc_args, 'dry-run', false);
   $force      = \WP_CLI\Utils\get_flag_value($assoc_args, 'force', false);
   $batch_size = (int) ($assoc_args['batch-size'] ?? 100);
   $sleep      = (int) ($assoc_args['sleep'] ?? 0);
   $mime_type  = $assoc_args['mime-type'] ?? null;
   $limit      = isset($assoc_args['limit']) ? (int) $assoc_args['limit'] : null;
   ```

2. Get total count via `$this->migrator->count_attachments($mime_type, $force)`
   - If zero: `WP_CLI::success('No files to offload.')` and return
   - If limit set and limit < total: use limit as total

3. Log start message: `WP_CLI::log("Found {$total} files to offload.")`
   - Log to file: `$this->log_to_file("START offload (batch_size={$batch_size}, dry_run=" . ($dry_run ? 'true' : 'false') . ")")`

4. **Dry-run branch:** If `$dry_run`:
   - Iterate through all batches, build items array with `$this->migrator->get_attachment_info($id)` for each
   - Respect $limit
   - Format as table: `\WP_CLI\Utils\format_items('table', $items, ['ID', 'Filename', 'Size', 'MIME'])`
   - `WP_CLI::log(sprintf("\n%d files would be uploaded.", count($items)))`
   - Return early (no uploads, no summary)

5. **Upload branch:** Initialize counters `$counters = ['success' => 0, 'failed' => 0, 'skipped' => 0]` and `$start_time = microtime(true)`

6. Register shutdown handler (see below)

7. Batch loop:
   ```
   $processed = 0;
   while (true) {
       $batch = $this->migrator->get_next_batch($batch_size, $mime_type, $force);
       if (empty($batch)) break;

       foreach ($batch as $id) {
           if ($limit !== null && $processed >= $limit) break 2;
           $processed++;

           $info = $this->migrator->get_attachment_info($id);
           $filename = $info['filename'];

           $result = $this->migrator->upload_attachment($id);

           if ($result['status'] === 'success') {
               $counters['success']++;
               WP_CLI::log(WP_CLI::colorize(sprintf("[%d/%d] Uploading %s... %%gOK%%n", $processed, $total, $filename)));
               $this->log_to_file("OK attachment:{$id} {$filename} ({$result['files']} files)");
           } elseif ($result['status'] === 'skip') {
               $counters['skipped']++;
               WP_CLI::log(WP_CLI::colorize(sprintf("[%d/%d] %s... %%ySkipped%%n - %s", $processed, $total, $filename, $result['error'])));
           } else {
               $counters['failed']++;
               WP_CLI::log(WP_CLI::colorize(sprintf("[%d/%d] Uploading %s... %%rFAILED%%n - %s", $processed, $total, $filename, $result['error'])));
               $this->log_to_file("FAIL attachment:{$id} {$filename} - {$result['error']}");
           }
       }

       $this->migrator->cleanup_memory();
       if ($sleep > 0) sleep($sleep);
   }
   ```

8. Print completion summary:
   ```php
   $elapsed = microtime(true) - $start_time;
   $speed = $counters['success'] > 0 ? round($counters['success'] / $elapsed, 1) : 0;

   WP_CLI::log('');
   WP_CLI::log('--- Migration Summary ---');
   WP_CLI::success(sprintf('%d files uploaded successfully', $counters['success']));
   if ($counters['failed'] > 0) {
       WP_CLI::warning(sprintf('%d files failed (see wp-content/ct-s3-migration.log)', $counters['failed']));
   }
   if ($counters['skipped'] > 0) {
       WP_CLI::log(sprintf('%d files skipped (already offloaded)', $counters['skipped']));
   }
   WP_CLI::log(sprintf('Elapsed: %s | Speed: %s files/sec', gmdate('H:i:s', (int) $elapsed), $speed));
   ```
   - Log summary to file: `$this->log_to_file("COMPLETE success={$counters['success']} failed={$counters['failed']} skipped={$counters['skipped']} elapsed={$elapsed}s")`

**Private helper methods:**

1. `private function log_to_file(string $message): void` - Append timestamped line to `WP_CONTENT_DIR . '/ct-s3-migration.log'` using `file_put_contents()` with `FILE_APPEND | LOCK_EX`

2. `private function register_shutdown_handler(array &$counters): void` - Reserve 256KB memory buffer, register shutdown function via `register_shutdown_function()` that frees buffer and prints partial summary on fatal error (E_ERROR, E_CORE_ERROR, E_COMPILE_ERROR). Use `fwrite(STDERR, ...)` for output. Include "Re-run 'wp ct-s3 offload' to resume" message.
